# Step 9D 规格：CSV、外部表格同步与写回统一标签校验

日期：2026-07-17  
状态：设计已获业务确认，待规格审阅  
分支：`feature/tag-step8-reply-tag-context`  
前置断点：`tag_skill_llm_step9c_breakpoint_069.md`

## 1. 目标

让 CSV 导入、外部表格同步、外部表格写回共用同一套标签解析和校验逻辑。

本步骤要保证：

1. 可识别的标签继续按现有规则进入正式客户数据。
2. 无法识别的标签原文完整写入现有 `unmatched_legacy_tag_values`，不猜测、不进入正式标签。
3. 普通客户资料不因一个无法识别的标签而整行丢失。
4. 外部同步或写回失败时，不覆盖客户本地已有的有效标签。
5. 重复导入、重复同步和重试保持幂等。

## 2. 范围

### 2.1 包含

- CSV 导入中的标签字段解析、校验和未匹配记录。
- `CustomerSyncScheduler` 外部表格逐行同步中的标签解析、校验和失败保护。
- `ManualSaveHandler`、`ExistingCustomerUpdater`、`NewCustomerRowCreator` 及写回重试链路中的标签校验。
- 复用 `TagDirectoryService`、`TagSelectionValidator` 和现有未匹配记录表。
- CSV、同步状态和写回结果中的未匹配/过滤结果表达。
- 后端单元测试、同步回归测试、表格写回回归测试和现有前端导入结果测试。

### 2.2 不包含

- 不新增 Flyway migration。
- 不修改现有 6 条 `system_tag_suggestions` 的原文、状态或关联数据。
- 不直接把跟进规则动作改成写入 `customer_tag_assignments`。
- 不改变 Step 8 回复标签上下文行为。
- 不开启 `llm.reply_generation.enabled` 或 `llm.profile_extraction.enabled`。
- 不把未匹配原文计入筛选、统计、回复或规则命中。
- 不进行模糊匹配或基于关键词的猜测映射。

## 3. 统一标签交换层

### 3.1 职责

新增一个只负责数据交换的应用服务边界。它不拥有客户标签写入事务，也不替代 `CustomerTagUpdateService` 的人工/系统标签分配；它负责：

- 根据字段绑定找到正式标签分类。
- 将外部原文拆分为单选或多选 token。
- 按“正式编码 -> 显示名称 -> 已配置同义词”的顺序做精确解析。
- 把解析出的正式编码交给 `TagSelectionValidator`，用途使用 `IMPORT`，业务依据包含来源和字段信息。
- 返回可保存的正式编码、未匹配原文和验证原因。
- 为入站保存和出站写回提供同一份结果结构。

### 3.2 解析规则

- 所有输入先 `trim`，空值不产生标签。
- 不做大小写模糊、包含、相似度或关键词猜测。
- 同一分类内重复 token 不静默去重，原样交给统一校验器，由校验器返回 `DUPLICATE_VALUES` 原因。
- 单选分类必须恰好得到一个可接受值。
- 多选分类可以得到多个可接受值；无法识别的 token 单独记录。
- 分类停用、分类合并、值停用、值合并或分类未开放 `manualEditEnabled` 时，交换层不得接受该值。

### 3.3 结果结构

交换层结果至少包含：

- 规范化后的可保存值（正式编码或源表可写入的正式名称）。
- 未匹配 token 列表及原始字段值。
- 分类字段、来源类型、来源记录标识。
- 是否允许继续保存普通字段。
- 单选保留当前值、多选保留有效部分、出站过滤等处理结论。

来源类型固定使用可读字符串：`CSV_IMPORT`、`EXTERNAL_SYNC`、`TABLE_WRITE`。

## 4. CSV 导入流程

1. 先完成 CSV 解析、手机号格式校验和同文件重复手机号校验。
2. 普通字段按现有导入逻辑处理；绑定标签字段交给统一交换层。
3. 单选标签无法识别时，不把该原文写入客户字段，也不替换客户当前有效标签。
4. 多选标签保留可识别值；无法识别 token 写入未匹配记录。
5. 该行的普通字段、可识别标签和未匹配记录在同一行事务中完成。
6. 手机号、重复行等现有行级错误仍然跳过；标签未匹配不增加 `skipped`。
7. `CsvImportResult` 增加未匹配数量及行号/字段摘要，保留现有 `created`、`updated`、`skipped` 和 `errors` 字段含义。

重复导入同一客户、字段和原文时，依靠现有未匹配记录唯一约束保持幂等。

## 5. 外部表格同步流程

1. `FieldMappingResolver` 先完成源字段到客户字段的映射。
2. 标签字段在生成 `Customer` 前经过统一交换层；无法识别的单选值不会进入 incoming customer。
3. `CustomerMergeEngine` 只合并普通字段和可接受标签值。单选无效值因此保留已有本地值。
4. 多选字段按现有来源行语义保存已识别值，未识别 token 只进入未匹配记录。
5. 客户合并、旧字段兼容同步和未匹配记录写入按行保持事务一致。
6. 标签目录停用或合并导致的校验失败属于数据质量结果，不把整行标记为同步失败。
7. 拉取失败、字段映射失败、客户保存失败或事务回滚进入现有 `sync_failure_log`，且不得留下部分客户标签更新。

外部同步失败时，下一轮仍按现有增量同步和失败记录机制工作；不会用失败行中的原文覆盖本地有效标签。

## 6. 外部表格写回流程

### 6.1 手工写回

- `ManualSaveHandler` 将源字段反解析为内部目标字段后，标签字段交给统一交换层。
- 可识别标签和普通字段继续写回；无法识别标签从本次写回 payload 中移除。
- 如果只剩无法识别标签，不发送空值覆盖外部表格，只记录未匹配结果并返回“无可写字段”。
- 本流程不修改本地正式客户标签。

### 6.2 自动写回和新客写回

- `ExistingCustomerUpdater`、`NewCustomerRowCreator` 生成内部字段后，统一交换层在 `toSourceFields` 前执行。
- 外部 `updateRow/createRow` 成功前，不改变本地标签或客户快照。
- 创建/更新成功后才记录外部行标识；创建失败时不插入带有新外部行标识的本地客户。
- 现有写回失败重试队列保留；队列重试必须重新读取标签目录并重新校验 payload。

## 7. 失败与保留规则

| 场景 | 普通字段 | 可识别标签 | 无法识别标签 | 本地有效标签 |
|---|---|---|---|---|
| CSV 行标签部分无效 | 继续导入 | 保存 | 写入未匹配记录 | 单选保留，多选按有效值处理 |
| 外部同步行标签部分无效 | 继续同步 | 合并 | 写入未匹配记录 | 单选无效时保留 |
| 外部同步拉取/保存失败 | 不更新 | 不更新 | 记同步失败 | 保持不变 |
| 外部表格写回部分无效 | 写回 | 写回 | 从 payload 移除并记录 | 不修改 |
| 外部表格写回网络失败 | 不确认成功 | 不确认成功 | 保留待处理 | 保持不变 |

校验失败不会抛出阻断整批任务的业务异常；映射缺失、数据库失败和远端失败仍沿用现有错误/失败队列机制。

## 8. 幂等、兼容和安全约束

- 未匹配记录沿用现有 `(customer_id, legacy_field, raw_value_hash)` 唯一约束，不生成重复记录。
- 已存在的 `CUSTOMER_FIELD` 历史兼容记录不被删除；新来源记录保留来源类型和来源行标识。
- 正式标签目录每次交换操作按最新快照校验；目录刷新失败时沿用 `TagDirectoryService` 的最近成功快照行为。
- 未匹配值不进入 `customer_tag_assignments`，不参与客户筛选、统计、回复和跟进规则。
- 不改变现有客户访问范围、管理员权限和写回权限检查。
- 不修改现有 6 条 PENDING 建议，不执行 merge、push 或创建 PR。

## 9. 验收测试

### 9.1 统一交换层

- 正式编码、显示名称、同义词的精确解析。
- 未知值、停用值、合并值、错误分类和值重复的拒绝原因。
- SINGLE 无效值保留当前值。
- MULTI 有效/无效混合只保存有效部分。
- 同一原文重复处理保持幂等。

### 9.2 CSV

- 普通资料与标签部分成功同一行提交。
- 单选无效标签不覆盖已有客户标签。
- 多选混合值写入有效部分并生成未匹配记录。
- 重复手机号、非法手机号继续沿用现有跳过逻辑。
- 导入结果包含未匹配数量和行号摘要。

### 9.3 外部同步

- 字段映射产生无效标签时，普通字段仍可保存。
- 拉取、映射、数据库保存异常时，客户和标签不产生部分更新。
- 同步失败记录保留源表、行号、手机号和原始行摘要。

### 9.4 外部写回

- 手工写回过滤无效标签但保留普通字段。
- 只包含无效标签时不发送空值覆盖外部行。
- 自动新客创建失败不插入本地外部行标识。
- 自动更新失败不修改本地有效标签。
- 重试重新按当前标签目录校验。
