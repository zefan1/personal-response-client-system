# 本地真实测试 Worklist

生成日期：2026-07-08  
目标：明确哪些事情需要用户配合，哪些可以由 Codex 代替完成，减少测试过程中来回卡点。

## A. 需要用户配合的事项

### A1. 外部服务与账号资料

- [ ] 提供真实 Skill/场景接口的 base URL、API key、测试模型/场景说明。
- [ ] 提供真实图像识别接口的 base URL、API key、模型名。
- [ ] 提供思考 LLM 接口的供应商选择、base URL、API key、模型名。
- [ ] 如果希望多 LLM 测试，至少提供 2 个模型或 2 套 provider 配置。
- [ ] 提供企微表格接口的 base URL、API key、测试表格信息。
- [ ] 确认这些 key 是否只能本机测试，还是未来服务器也能使用。

### A2. 业务测试样本

- [ ] 提供 5-10 张真实或脱敏聊天截图，用于图像识别测试。
- [ ] 提供 10 条典型聊天文本，用于思考 LLM 回复建议测试。
- [ ] 提供 10 条客户样例：手机号、昵称、来源、阶段、标签、跟进状态。
- [ ] 提供企微表格测试表或测试空间，避免污染真实业务数据。
- [ ] 提供期望回复风格：专业、亲切、销售转化强度、禁用话术。
- [ ] 提供客户分层/标签规则中最关键的业务判断标准。

### A3. 产品决策

- [ ] 确认思考 LLM 是否与 Skill 接口并存，还是逐步替代部分 Skill 能力。
- [ ] 确认多 LLM 配置粒度：全局默认、按场景选择、按账号选择，还是按 A/B 测试选择。
- [ ] 确认管理后台是否需要展示每次 LLM 调用成本、耗时、命中模型。
- [ ] 确认失败降级策略：换备用模型、返回模板回复、提示人工处理，优先级如何排。
- [ ] 确认侧边栏正式交付形态：安装包、免安装包、还是先内部 dev 运行。
- [ ] 确认服务器上线时间与目标环境：云服务器、内网服务器、还是本机长期运行。

### A4. 本机/服务器访问条件

- [ ] 如果需要我操作你的真实 key，请确认是否可以写入本地数据库配置表。
- [ ] 如果外部接口有 IP 白名单，请确认本机公网 IP 是否已加入。
- [ ] 后续上服务器时，提供服务器系统、登录方式、域名、端口开放策略。
- [ ] 后续需要正式发布桌面端时，确认是否购买/准备 Windows 代码签名证书。

当前结论：

## C. 推荐分工顺序

1. Codex 先让当前已开发功能在本地动起来：后端、数据库、Redis、侧边栏、后台基础链路。
2. Codex 检查本地环境、数据库、桌面端配置，并修复首要阻塞项。
3. Codex 跑 mock/基础链路基线，输出“能用/不能用/为什么不能用”的问题清单。
4. 用户提供本地测试用真实接口 key 和测试样本。
5. Codex 已补齐思考 LLM 多模型、五个业务场景路由、调用日志和本地主备降级验收。
6. 用户提供真实 key 后，Codex 接入真实图像识别、思考 LLM、Skill、表格接口，并跑 live 验收。
7. 用户确认回复风格、业务规则、客户样本是否符合预期。
8. Codex 跑侧边栏完整测试并修复问题。
9. Codex 跑管理后台完整测试并修复问题。
10. 用户确认未来服务器环境与桌面端发布方式。
11. Codex 输出服务器部署方案和侧边栏发布方案。
