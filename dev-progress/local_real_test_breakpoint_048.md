# 断点 048：管理后台真机反馈修复批次 1

时间：2026-07-10 11:05  
状态：已完成

## 本轮用户反馈

- Skill/LLM/跟进规则里的“优先级”缺少解释，不知道应该填多少。
- 运营 B 配置中心看不懂。
- 速搜正文不能插入客户档案变量；切换内容类型后表单没有明显变化。
- 报错提示仍有英文，需要面向人展示中文。
- 新建账号登录后能看到其他账号数据，存在越权风险。
- 后台重置密码后，侧边栏旧登录态和旧密码仍可能继续使用。
- 跟进规则线索类型缺少“全部客资”。
- 标签、分析、审计、健康等页面存在不可读、不可理解的问题。
- 发布系统公告后，侧边栏工作台没有公告提示。

## 已完成修复

- 新增 `accounts.token_version` 迁移，用于重置密码、停用、改角色后让旧 access token 和 WebSocket 立即失效。
- 登录、刷新 token、JWT 认证、WebSocket 握手均接入 `tokenVersion` 校验，旧 token 会提示“账号权限或密码已变更，请重新登录”。
- 账号重置密码、启停、角色/直属组长修改后，会递增 `token_version` 并撤销 refresh token。
- 新增客户访问控制：管理员看全部；管家只看分配给自己的客户；组长看自己和直属启用管家的客户。
- 客户搜索、客户档案、今日跟进、手动编辑、保存到表格、档案建议队列均接入客户访问判断。
- 管理后台跟进规则支持“全部客资”，运行时先匹配具体线索类型，再回退到通用规则。
- 管理后台速搜内容支持插入客户档案变量，例如 `{{nickname}}`、`{{store}}`、`{{customerStage}}`；侧边栏复制时会用当前客户档案值替换。
- 速搜内容类型切换后，正文/图片/小程序等字段的标签、提示和说明会跟着变化。
- 工作台新增主动加载 `/api/v1/notices/active`，并能渲染系统公告卡片；WebSocket 公告推送不会重复显示。
- 前端 API 超时、网络失败、后台常见异常统一转为中文提示。
- 管理后台增加优先级说明、配置顺序说明、字段中文映射、标签分类中文映射、分析摘要中文化、审计动作分组展示。
- 修复 `SkillRuntimeRouter` 通用路由编译错误：使用方法参数 `leadType` 查询绑定，而不是不存在的 `normalizedLeadType`。
- 更新测试预期：当前产品行为应显示“全部客资”、公告应显示在工作台、错误应中文化。
