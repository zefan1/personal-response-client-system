# 本地真机复测 Checklist（管理后台 + 侧边栏）

时间：2026-07-10  
用途：给用户边测边勾选，确认断点 049 后的 6 个核心问题是否已经可用。

## 测试前准备

- [ ] 后端正在运行：`http://localhost:8080`
- [ ] 前端正在运行：`http://127.0.0.1:5173/#/admin`
- [ ] 后端已重启到最新代码；如果顶部还显示“网络连接失败”，先刷新页面
- [ ] 管理后台已重新登录，不使用旧登录态
- [ ] 侧边栏已登录同一个本地后端

> 如果后台顶部出现“网络连接失败”，先点击“退出”，用 `admin/admin123` 重新登录。旧 token 被重置密码/权限变更后会失效，这是正常安全行为。
> 本轮已确认：截图监听代码只读取剪贴板图片和屏幕图像，没有调用切换输入法的接口。若截图后仍自动变英文，请在第 6 项后记录触发方式，优先排查 Windows 截图快捷键或前台聊天软件的输入焦点切换。

## 1. 账号创建与管家直属组长校验

- [ ] 打开管理后台 -> `E 账号与权限`
- [ ] 点击 `新增账号`
- [ ] 角色选择 `管家`
- [ ] 不选择直属组长，填写手机号、初始密码、姓名后点击保存
- [ ] 预期：不能保存，提示 `管家账号必须选择直属组长`
- [ ] 再创建一个 `组长` 账号
- [ ] 预期：组长可以不选择直属组长并保存成功
- [ ] 再创建一个 `管家` 账号，并选择刚创建的组长
- [ ] 预期：管家保存成功，列表里显示直属组长

备注：

## 2. 密码重置后旧密码与旧登录态失效

- [ ] 用刚创建的管家账号登录侧边栏
- [ ] 确认侧边栏能进入工作台
- [ ] 回到管理后台 -> `E 账号与权限`
- [ ] 找到该管家账号，点击 `重置密码`
- [ ] 设置一个新密码并保存
- [ ] 回到侧边栏刷新或操作任意需要接口的功能
- [ ] 预期：旧登录态失效，自动返回登录页并提示 `登录已过期，请重新登录`
- [ ] 用旧密码登录
- [ ] 预期：登录失败
- [ ] 用新密码登录
- [ ] 预期：登录成功

备注：

## 3. 客户数据隔离

- [ ] 用管理员账号在管理后台确认 `1111` 客户能被搜索到
- [ ] 用新建的空管家账号登录侧边栏
- [ ] 打开 `客户档案`
- [ ] 搜索 `1111`
- [ ] 预期：搜索不到不属于该管家的客户
- [ ] 打开 `工作台` 或 `跟进列表`
- [ ] 预期：只显示该管家自己的客户；如果没有分配客户，应为空状态

备注：

## 4. 系统公告展示

- [ ] 管理后台打开 `J 系统公告`
- [ ] 点击 `新增公告`
- [ ] 填写标题和内容，发布方式选择 `立即发布`
- [ ] 保存
- [ ] 预期：公告列表出现该公告，状态为 `已发布`
- [ ] 指标卡 `生效中` 数量会增加
- [ ] 打开侧边栏 `工作台`
- [ ] 预期：能看到刚发布的公告提示

备注：

## 5. 速搜变量替换

- [ ] 管理后台打开 `D 速搜内容管理`
- [ ] 点击 `新增内容`
- [ ] 线索类型选择 `全部客资`
- [ ] 在正文里点击变量按钮，例如 `客户昵称`、`意向门店`
- [ ] 预期：正文插入 `{{nickname}}`、`{{intendedStore}}` 等变量
- [ ] 保存后回到侧边栏，打开对应客户档案
- [ ] 打开速搜并复制该内容
- [ ] 预期：复制出来的内容会按当前客户档案替换变量

备注：

## 6. 跟进规则全部客资与优先级说明

- [ ] 管理后台打开 `F 跟进规则引擎配置`
- [ ] 点击 `新增规则`
- [ ] 线索类型选择 `全部客资`
- [ ] 查看优先级说明
- [ ] 预期：说明里能看到 `数字越大越先执行`
- [ ] 预期：说明里能看到建议值：紧急提醒 90、普通提醒 60、观察类 30
- [ ] 保存一个临时规则
- [ ] 预期：列表展示该规则，线索类型不限于某一种客资
- [ ] 测完后删除或停用临时规则
